Requisitos de Software:
1. Funcionales:
-El sistema debe permitir al cliente ingresar una solicitud de recolección especificando el tipo de material y cantidad.
-El sistema debe permitir al cliente registrar la ubicación geográfica de la solicitud
-El sistema debe validar automáticamente que la ubicación ingresada por el Cliente se encuentre dentro del área de recolección permitida.
-El sistema debe permitir al reciclador generar un reporte del estado de la solicitud (Por ejemplo: retirada o no realizada), incluyendo observaciones.
-El sistema debe permitir al reciclador emitir un comprobante de retiro para el cliente tras una recolección exitosa.
-El sistema debe permitir al coordinador de ruta visualizar las solicitudes pendientes en un mapa.
-El sistema debe permitir al coordinador de ruta agrupar solicitudes para generar rutas de recolección.
-El sistema debe permitir a los usuarios (los clientes, recicladores y coordinadores) gestionar sus credenciales de acceso (Inicio o cierre de sesión).

2. No funcionales:
-Usabilidad: La aplicación debe poseer una interfaz adaptada al perfil del usuario (vistas simplificadas para los recicladores en terreno y vistas de gestión para los coordinadores).
-Portabilidad: La aplicación debe ser accesible tanto desde dispositivos móviles (Android/iOS) como navegadores web de escritorio.
-Accesibilidad: La aplicación debe cumplir con pautas de accesibilidad web para usuarios con discapacidades visuales o motoras.
-Disponibilidad: El sistema debe estar disponible para el ingreso de solicitudes en los horarios definidos por la cooperativa (24/7 o horario hábil según la regla que posea el negocio).
-Seguridad: Las contraseñas de los usuarios deben almacenarse de forma encriptada y el sistema debe validar la identidad antes de permitir el acceso a datos sensibles.
